Detailed Description

Stores the state of FragmentedSystem class.

Right now we just strore the state as a series of maps. It's assumed that users will create a PIMPL and then create a FragmentedSystem class by wrapping that PIPML. This works for the moment, but down the road we may want to allow users to add frags/caps etc. dynamically which is difficult with this data layout. If that happens the PIMPL should be made into an abstract base class and additional implementations added.
